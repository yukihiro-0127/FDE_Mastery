小売業AI導入の現実: 予測精度80%でも大きな価値

📋 ケース概要

顧客: 全国展開するコンビニチェーン（店舗数5,000店、年商1兆円）

課題: 商品の発注量が適切でなく、廃棄ロスと機会損失が発生。月間廃棄ロス5億円、機会損失2億円。

要望: AIで需要を予測し、在庫を最適化したい。目標は廃棄ロス50%削減。

予算: 初期投資1億円、年間運用コスト2,000万円

期間: PoC 3ヶ月、本番導入 1年

💡 小売業AIの本質的な課題

小売業では、「過去データだけでは予測できない」。天気、イベント、競合の動きなど、外部要因が大きく影響する。

さらに、商品ごとに予測精度が異なる。定番商品は予測しやすいが、新商品や季節商品は難しい。

小売業特有の3つの制約:

  • 外部要因の影響: 天気、イベント、競合の動きが需要に大きく影響
  • 商品の多様性: 定番商品、季節商品、新商品で予測精度が異なる
  • リアルタイム性: 発注は毎日行うため、予測も毎日更新が必要

FDEとしての提案: 時系列予測 + 外部データ統合

システム構成

【入力データ】
内部データ:
- 過去の販売データ（2年分、商品別・店舗別・時間帯別）
- 在庫データ（現在の在庫量）
- 価格データ（定価、セール価格）

外部データ:
- 天気予報（気温、降水確率、湿度）
- イベント情報（祝日、地域イベント、スポーツ大会）
- 競合情報（近隣店舗のセール情報）
- SNSトレンド（話題の商品）
    ↓
【前処理】
データクレンジング、欠損値補完、正規化
特徴量エンジニアリング（曜日、時間帯、季節性）
    ↓
【AI予測】
翌日の需要予測（商品別・店舗別）
時系列予測モデル（LSTM + XGBoost）
    ↓
【発注最適化】
予測に基づいて自動発注
安全在庫を考慮した発注量計算
    ↓
【結果】
廃棄ロス削減、機会損失削減、在庫回転率向上

なぜこの構成なのか

時系列予測（LSTM）

役割: 過去のトレンドを学習

メリット:

  • 季節性、トレンドを捉える
  • 長期的なパターンを学習
  • 時系列データに最適

適用商品: 定番商品、季節商品

勾配ブースティング（XGBoost）

役割: 外部要因を考慮

メリット:

  • 天気、イベントなどの外部要因を学習
  • 高精度
  • 特徴量の重要度を可視化

適用商品: 全商品

技術選定の詳細

項目 選定技術 理由
時系列予測 LSTM（Long Short-Term Memory） 季節性、トレンドを捉える、長期的なパターンを学習
外部要因考慮 XGBoost 天気、イベントなどの外部要因を学習、高精度
データベース PostgreSQL + TimescaleDB 時系列データに最適、高速クエリ
クラウド AWS（S3 + SageMaker） スケーラビリティ、機械学習機能が充実

実装のポイント

1. 外部データの統合

なぜ外部データが重要なのか

理由: 過去データだけでは予測できない要因が多い

例:

  • 気温が30度を超えるとアイスの売上が2倍になる
  • 雨の日は傘の売上が5倍になる
  • 祝日は来客数が1.5倍になる

外部データ1: 天気

取得方法: 気象庁API、OpenWeatherMap API

データ項目:

  • 気温（最高気温、最低気温）
  • 降水確率
  • 湿度
  • 風速

影響例:

  • 気温30度以上 → アイス売上2倍
  • 降水確率80%以上 → 傘売上5倍
  • 湿度80%以上 → 飲料売上1.5倍

外部データ2: イベント

取得方法: カレンダーAPI、地域イベント情報

データ項目:

  • 祝日
  • 地域イベント（祭り、花火大会）
  • スポーツ大会（野球、サッカー）

影響例:

  • 祝日 → 来客数1.5倍
  • 地域イベント → 来客数2倍
  • スポーツ大会 → ビール売上3倍

外部データ3: 競合

取得方法: 競合店舗の調査、価格比較サイト

データ項目:

  • 近隣店舗のセール情報
  • 価格情報
  • 新商品情報

影響例:

  • 競合がセール → 来客数20%減
  • 競合が新商品 → 類似商品売上10%減

外部データ4: SNSトレンド

取得方法: Twitter API、Google Trends

データ項目:

  • 話題の商品
  • トレンドワード
  • インフルエンサーの投稿

影響例:

  • SNSで話題 → 売上5倍
  • インフルエンサーが紹介 → 売上10倍

2. 商品カテゴリ別の予測

定番商品（おにぎり、飲料）

特徴: 毎日売れる、季節性が少ない

予測精度: 90%

予測手法: LSTM（過去のトレンドを学習）

外部要因: 天気（気温、降水確率）

季節商品（アイス、おでん）

特徴: 季節によって売上が大きく変動

予測精度: 70%

予測手法: LSTM + XGBoost（季節性 + 外部要因）

外部要因: 天気（気温）、季節

新商品

特徴: 過去データがない

予測精度: 50%

予測手法: 類似商品のデータで予測

外部要因: SNSトレンド、広告効果

トレンド商品

特徴: SNSで話題、短期間で売上が急増

予測精度: 60%

予測手法: SNSトレンド分析 + XGBoost

外部要因: SNSトレンド、インフルエンサー

3. 発注最適化

発注量の計算

計算式:

発注量 = 予測需要 + 安全在庫 - 現在在庫

安全在庫 = 予測誤差 × 安全係数

予測誤差 = 過去の予測と実績の差の標準偏差

安全係数 = 1.65（95%の確率で在庫切れを防ぐ）

廃棄ロスと機会損失のトレードオフ

問題: 発注量を増やすと廃棄ロスが増え、減らすと機会損失が増える

解決策: 最適化問題として解く

最小化: 廃棄ロスコスト + 機会損失コスト

廃棄ロスコスト = (発注量 - 実需要) × 原価

機会損失コスト = (実需要 - 発注量) × 利益

商品別の発注戦略

定番商品: 安全在庫を多めに（機会損失を防ぐ）

季節商品: 安全在庫を少なめに（廃棄ロスを防ぐ）

新商品: 少量発注で様子見（リスクを最小化）

リアルタイム調整

方法: 1日3回予測を更新

  • 朝6時: 当日の需要予測
  • 昼12時: 午後の需要予測（午前の実績を反映）
  • 夕方18時: 夜の需要予測（午後の実績を反映）

失敗事例から学ぶ

❌ 失敗事例1: 過去データのみで予測

問題: 天気、イベントを考慮せず、予測精度が低い

詳細:

  • 過去の販売データのみで予測
  • 天気、イベントを考慮せず
  • 予測精度60%と低い
  • 廃棄ロスが減らない

教訓: 外部データ（天気、イベント）を統合する。

❌ 失敗事例2: 全商品を同じ手法で予測

問題: 定番商品と季節商品を同じ手法で予測し、精度が低い

詳細:

  • 全商品をLSTMで予測
  • 季節商品の予測精度が50%と低い
  • 廃棄ロスが減らない

教訓: 商品カテゴリ別に予測手法を変える。

❌ 失敗事例3: 安全在庫を考慮せず

問題: 予測通りに発注し、在庫切れが多発

詳細:

  • 予測需要 = 発注量
  • 予測誤差を考慮せず
  • 在庫切れが多発
  • 機会損失が増加

教訓: 安全在庫を考慮した発注量計算。

❌ 失敗事例4: 一度に全店舗に展開

問題: 予測精度が低く、全店舗で廃棄ロスが増加

詳細:

  • PoCなしで全店舗に展開
  • 予測精度が低い
  • 全店舗で廃棄ロスが増加
  • 損失10億円

教訓: 段階的導入（PoC → パイロット → 本番）。
